# Data catalog

Claros is a generic adapter over the EIA APIv2, one uniform faceted API. Any of the 232 datasets below can become a Claros feed by mapping it to a `feed_id`. 36 are already live (see [live feeds](/docs/feeds)); the rest are one mapping row away.

## How to read this catalog

Every EIA dataset is addressed the same way:

- **Route**: the dataset path, for example `petroleum/pri/spt`. This is the `{route}` in the EIA endpoint `/v2/{route}/data`.
- **Frequencies**: the time resolutions the dataset supports (daily, weekly, monthly, quarterly, annual, hourly). You pick exactly one.
- **Columns**: the numeric columns the dataset returns, for example `value`, `price`, `quantity`. You read one as the feed value.
- **Facets**: the filters. Each facet has an `id` (for example `series`, `duoarea`, `stateid`, `fueltype`) and you choose a value for it to pin down one series.

A Claros feed is exactly this tuple: `{ route, frequency, data_col, facets }`. For example `EIA.PET.PRICE.WTI.DAILY` is route `petroleum/pri/spt`, frequency `daily`, column `value`, facet `series=RWTC`.

<Callout type="info">
Rows marked **Live** already have a Claros `feed_id`. The rest are available to map: same adapter, one more row in the feed catalog.
</Callout>
